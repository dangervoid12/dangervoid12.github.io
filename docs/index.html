Technical Scope

Drawing and Geometry

  • Integrated drawing tools allow production geometry to be created and corrected inside the same project.
  • DXF and DWG based work can be prepared without moving to another CAD system for small and mid-size edits.
  • Layer-aware editing helps separate contours, drills, marks, bend lines, and process-specific entities.
  • Geometry cleanup supports preparation before nesting or machining when imported files are not ready for output.
  • Drawing work can continue directly into export or into the downstream CAM flow when the selected license allows it.

Files and Project Data

  • The Files tab acts as the project entry point for loaded parts, source drawings, and grouped job content.
  • Project data stays together across tabs so imported files, materials, methods, and machines belong to one workflow.
  • Saved project state is designed to reopen with the same production context instead of isolated tab data.
  • File preparation supports both design-driven projects and production-driven projects that begin from customer geometry.
  • Export and report output can be produced from the same project data without rebuilding the job elsewhere.

Materials and Cutting Data

  • Material definitions keep sheet data, process settings, and cost-related values available to the production workflow.
  • Feeds and speeds information can be loaded with the project so cutting guidance is available when machining is prepared.
  • Material setup supports repeated shop workflows where the same stock types return across many jobs.
  • Centralized cutting data reduces manual re-entry when users move from files to nesting to machining.
  • Production decisions stay closer to actual machine and tool conditions because settings are stored in the software context.

Nesting and Machining

  • Nesting arranges project parts on sheets as part of the same production flow instead of requiring an external optimizer.
  • Machining logic builds operations after geometry and material preparation, which keeps the process sequencing consistent.
  • Toolpaths can be prepared from imported files, from nested layouts, or from specialized module outputs.
  • CAM preparation stays connected to the machine context so output decisions remain tied to real production targets.
  • Report generation can reflect nesting and machining results from the current job rather than from disconnected exports.

Specialized Production Modules

  • Sheet Metal supports unfolding-oriented work for fabrication tasks that need flat pattern preparation.
  • Furniture tools support cabinet and layout-oriented workflows for panel-based production projects.
  • Heightmap tools extend the software beyond flat geometry into relief and surface-based machining work.
  • Specialized modules remain inside one application so users do not have to rebuild the same project in separate systems.
  • Each module can feed forward into later stages such as export, nesting, machining, generation, and reporting.

Machines, Generation, and Output

  • Machine profiles store production-specific settings that influence how output is generated for the current project.
  • The Generation tab works against the active machine so postprocessing stays aligned with the selected output target.
  • Full license projects are limited to one machine definition, which fits a single-machine production setup.
  • Multi-machine projects support more than one machine definition and allow machine selection inside Generation.
  • Output, export, and reporting stay connected to the same project state, which reduces mismatches between setup and result.

Built for Real Shops

Software Scope

GcodePlanner keeps design and production work in one project.

  • CAD-focused work: drawing and preparation are available in one place.
  • CAM-focused work: nesting and machining stay in the same project.
  • Full projects: output stays aware of the machine selected for generation.

Machine-Aware Generation

The Generation tab works with the current machine context so postprocessing stays aligned with the selected machine setup.

  • Full license: one machine can be added to the project.
  • Multi-machine license: multiple machines can be added to the same project.
  • Demo license: generation is disabled and drawings cannot be saved.

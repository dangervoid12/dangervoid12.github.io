License Logic

GcodePlanner is licensed by workflow depth, not only by feature count. That means each license is meant to match how a shop actually works inside the software. Demo is for review and evaluation. CAD is for users who prepare geometry, furniture layouts, and sheet metal design data without running the production side. CAM is for users who already have geometry and need to move through materials, nesting, machining, generation, export, and report output. Full combines the design and production paths inside one project but keeps the machine model limited to a single configured machine. Multi-machine extends the same complete workflow with more than one machine in the project and a current-machine selection in the Generation tab. This keeps licensing tied to real operational scope instead of unlocking unrelated tools without context.

The practical effect is that access changes by stage of work. A design-focused seat can stay centered on drawing, furniture, sheet metal, files, and export. A production-focused seat can stay centered on files, materials, nesting, machining, generation, export, and report output. A full production office can keep everything in one place and then decide whether the job belongs to one machine or to several machines. This is why the machine rules are part of the license logic and not only part of the machine tab.

What Changes by License

Full license users can move from design tabs to production tabs without leaving the same project. The important limit is machine ownership inside that project. One machine can be added, maintained, and used as the active output target, which fits a workshop that programs for one physical machine or one stable postprocessor setup.

Multi-machine keeps the same complete workflow but removes the one-machine project limit. More than one machine can be added, each machine can carry its own setup, and the current machine can be selected in Generation when output is prepared. That is useful when the same project may run on different routers, cells, or output configurations.

CAD and CAM licenses stay narrower by design. CAD supports authoring and preparation work around Drawing, Furniture, Sheet Metal, Files, and Export. CAM supports production work around Files, Materials, Nesting, Machining, Generation, Export, and Report. Demo is intentionally restrictive so evaluation can happen without creating production output or saving finished drawing work.
